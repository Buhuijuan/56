# MyCampus 前端调用后端说明

本文档用于告诉前端如何调用已经部署到服务器上的 MyCampus Spring Boot 后端。

适用对象：

- Unity 前端
- Web 前端
- 测试同学
- 联调同学

本文档重点说明：

- 后端部署后前端应该请求哪个地址
- 哪些接口需要登录，哪些不需要
- 每个接口的作用
- 每个接口的请求方式、请求体、请求头
- 一个完整的调用流程示例

## 1. 后端基础信息

假设你的后端已经部署到服务器，并且对外地址为：

```text
https://api.your-domain.com
```

那么前端所有请求都应基于这个 `Base URL`：

```text
https://api.your-domain.com
```

例如：

- 登录接口：`POST https://api.your-domain.com/api/auth/login`
- 获取玩家信息：`GET https://api.your-domain.com/api/player/me`

如果你当前还没有域名，也可以临时用服务器 IP：

```text
http://服务器IP
```

例如：

```text
http://123.123.123.123/api/auth/login
```

## 2. 请求规则

### 2.1 请求头

所有 JSON 接口都建议带：

```http
Content-Type: application/json
Accept: application/json
```

需要登录的接口还要带：

```http
Authorization: Bearer <token>
```

其中 `<token>` 来自登录接口 `POST /api/auth/login` 返回值里的 `data.token`。

### 2.2 响应格式

绝大部分接口都遵循统一格式：

```json
{
  "success": true,
  "data": {},
  "message": null
}
```

失败时通常是：

```json
{
  "success": false,
  "data": null,
  "message": "错误原因"
}
```

### 2.3 HTTP 状态码

常见情况：

- `200`：请求成功
- `400`：参数错误，或者业务条件不满足
- `401`：未登录或 token 无效
- `403`：无权限
- `404`：资源不存在
- `500`：服务端异常

### 2.4 鉴权规则

不需要登录的接口：

- `POST /api/auth/send-code`
- `POST /api/auth/register`
- `POST /api/auth/login`
- `POST /api/auth/reset-password`
- `POST /api/campus/qa`
- `GET /actuator/health`

其余大多数 `/api/**` 接口都需要先登录。

## 3. 推荐的前端调用顺序

前端正常接入时，推荐按下面顺序工作：

1. 用户进入登录页
2. 调用 `POST /api/auth/login`
3. 保存返回的 `token`
4. 调用 `GET /api/player/me`
5. 如果已经有当前角色，继续调用 `GET /api/game/home`
6. 进入主菜单或游戏场景
7. 游戏运行过程中按模块调用任务、签到、成长、活动等接口

如果是新用户：

1. 调用 `POST /api/auth/send-code`
2. 调用 `POST /api/auth/register`
3. 调用 `POST /api/auth/login`
4. 调用 `POST /api/player/roles` 创建角色
5. 调用 `GET /api/game/home`

## 5. 各接口详细说明

## 5.1 认证模块

### 5.1.1 发送验证码

接口：

```http
POST /api/auth/send-code
```

是否需要登录：

- 不需要

作用：

- 给指定邮箱发送验证码
- 用于注册
- 用于找回密码

请求体：

```json
{
  "mailbox": "test@example.com"
}
```

成功响应示例：

```json
{
  "success": true,
  "data": {
    "mailbox": "test@example.com",
    "sent": true
  },
  "message": null
}
```

前端用途：

- 点击“发送验证码”按钮时调用
- 成功后开启倒计时

### 5.1.2 注册

接口：

```http
POST /api/auth/register
```

是否需要登录：

- 不需要

作用：

- 使用邮箱、密码、验证码创建账号

请求体：

```json
{
  "mailbox": "test@example.com",
  "password": "12345678",
  "verificationCode": "123456"
}
```

成功响应示例：

```json
{
  "success": true,
  "data": {
    "accountId": 1,
    "accountCode": "A000001",
    "mailbox": "test@example.com",
    "currentRoleId": null,
    "token": "jwt-token-string"
  },
  "message": null
}
```

前端用途：

- 完成账号创建
- 当前项目更推荐注册成功后再主动调用一次登录接口

### 5.1.3 登录

接口：

```http
POST /api/auth/login
```

是否需要登录：

- 不需要

作用：

- 账号密码登录
- 获取后续所有鉴权接口要用的 JWT token

请求体：

```json
{
  "mailbox": "test@example.com",
  "password": "12345678"
}
```

成功响应示例：

```json
{
  "success": true,
  "data": {
    "accountId": 1,
    "accountCode": "A000001",
    "mailbox": "test@example.com",
    "currentRoleId": 10,
    "token": "jwt-token-string"
  },
  "message": null
}
```

前端用途：

- 保存 `data.token`
- 后续请求统一带上 `Authorization: Bearer <token>`
- 登录成功后立即请求 `GET /api/player/me`

### 5.1.4 重置密码

接口：

```http
POST /api/auth/reset-password
```

是否需要登录：

- 不需要

作用：

- 通过邮箱验证码重置密码

请求体：

```json
{
  "mailbox": "test@example.com",
  "verificationCode": "123456",
  "newPassword": "newPassword123"
}
```

## 5.2 玩家模块

### 5.2.1 获取当前玩家信息

接口：

```http
GET /api/player/me
```

是否需要登录：

- 需要

作用：

- 获取当前账号信息
- 获取账号下的角色列表
- 获取当前角色 ID

请求头示例：

```http
Authorization: Bearer jwt-token-string
```

前端用途：

- 恢复会话
- 判断是否已有角色
- 菜单页初始化

### 5.2.2 创建角色

接口：

```http
POST /api/player/roles
```

是否需要登录：

- 需要

作用：

- 创建新角色
- 初始化角色档案、成长、任务、签到、活动等初始数据

请求体：

```json
{
  "schoolId": 1,
  "campusName": "安心大学",
  "nickName": "小明",
  "characterId": 1
}
```

字段说明：

- `schoolId`：学校 ID，可选
- `campusName`：校区名或学校展示名，可选
- `nickName`：昵称，必填
- `characterId`：角色形象 ID，必填

前端用途：

- 新用户第一次进入游戏时创建角色
- 调用成功后建议马上请求 `GET /api/game/home`

### 5.2.3 切换当前角色

接口：

```http
PUT /api/player/current-role
```

是否需要登录：

- 需要

作用：

- 切换当前账号下的活跃角色

请求体：

```json
{
  "roleId": 10
}
```

前端用途：

- 多角色切换
- 成功后建议重新拉一次 `GET /api/game/home`

## 5.4 任务模块

### 5.4.1 获取任务总览

接口：

```http
GET /api/tasks
```

作用：

- 获取当前任务列表
- 获取可接、进行中、已完成、已领奖任务
- 获取当前主线任务

前端用途：

- 打开任务面板时刷新

### 5.4.2 获取单个任务详情

接口：

```http
GET /api/tasks/{taskCode}
```

示例：

```http
GET /api/tasks/M_1_1
```

作用：

- 获取指定任务详情

### 5.4.3 接受任务

接口：

```http
POST /api/tasks/{taskCode}/accept
```

作用：

- 把可接任务变为进行中

### 5.4.4 直接推进任务

接口：

```http
POST /api/tasks/{taskCode}/progress
```

作用：

- 直接对某个任务推进进度
- 一般更推荐前端上报任务事件，而不是手动推进

请求体：

```json
{
  "eventType": "NPC_DIALOGUE",
  "targetType": "npc_dialogue",
  "targetId": 1003,
  "increment": 1,
  "extra": {}
}
```

### 5.4.5 领取任务奖励

接口：

```http
POST /api/tasks/{taskCode}/claim
```

作用：

- 领取任务奖励

### 5.4.6 获取当前主线任务

接口：

```http
GET /api/tasks/current/main
```

作用：

- 获取当前主线任务

### 5.4.7 获取章节状态

接口：

```http
GET /api/tasks/chapters
```

作用：

- 获取主线章节完成状态

### 5.4.8 上报任务事件

接口：

```http
POST /api/tasks/events
```

作用：

- 前端只上报玩家行为
- 后端负责判断任务是否推进、完成、解锁

请求体示例：

```json
{
  "eventType": "ARRIVE_BUILDING",
  "targetType": "arrive_building",
  "targetId": 2001,
  "increment": 1,
  "currentPosX": 10,
  "currentPosY": 0,
  "currentPosZ": 20,
  "distanceToTarget": 5,
  "success": true,
  "extra": {}
}
```

支持的关键事件类型：

- `AI_DIALOGUE`
- `ARRIVE_BUILDING`
- `LANDMARK_VISIT`
- `NPC_DIALOGUE`
- `PHOTO_CHECKIN`
- `BIKE_STATION_VISIT`
- `BIKE_TRIAL_DISTANCE`

前端建议：

- 任务系统优先走这个接口
- 不要前端自己判断任务是否完成

## 6. 前端最常用的完整调用示例

下面给一个最常见的完整流程：

- 用户登录
- 获取玩家信息
- 获取首页数据
- 上报一次任务事件
- 查询当日打卡信息

### 6.1 第一步：登录

```bash
curl -X POST "https://api.your-domain.com/api/auth/login" \
  -H "Content-Type: application/json" \
  -d '{
    "mailbox": "test@example.com",
    "password": "12345678"
  }'
```

假设返回：

```json
{
  "success": true,
  "data": {
    "accountId": 1,
    "accountCode": "A000001",
    "mailbox": "test@example.com",
    "currentRoleId": 10,
    "token": "jwt-token-string"
  },
  "message": null
}
```

前端需要保存：

- `token`
- `accountId`
- `currentRoleId`

### 6.2 第二步：获取玩家信息

```bash
curl "https://api.your-domain.com/api/player/me" \
  -H "Authorization: Bearer jwt-token-string"
```

作用：

- 拉账号信息
- 拉角色列表

### 6.3 第三步：获取首页聚合数据

```bash
curl "https://api.your-domain.com/api/game/home" \
  -H "Authorization: Bearer jwt-token-string"
```

作用：

- 初始化主菜单
- 初始化主场景缓存

## 7. Unity 侧调用示例

下面给一个 Unity `UnityWebRequest` 的调用例子，以登录接口为例：

```csharp
using System.Collections;
using System.Text;
using UnityEngine;
using UnityEngine.Networking;

public class BackendExample : MonoBehaviour
{
    private const string BaseUrl = "https://api.your-domain.com";

    [System.Serializable]
    public class LoginRequest
    {
        public string mailbox;
        public string password;
    }

    public IEnumerator Login()
    {
        var body = new LoginRequest
        {
            mailbox = "test@example.com",
            password = "12345678"
        };

        string json = JsonUtility.ToJson(body);
        using var request = new UnityWebRequest(BaseUrl + "/api/auth/login", "POST");
        byte[] bodyRaw = Encoding.UTF8.GetBytes(json);
        request.uploadHandler = new UploadHandlerRaw(bodyRaw);
        request.downloadHandler = new DownloadHandlerBuffer();
        request.SetRequestHeader("Content-Type", "application/json");
        request.SetRequestHeader("Accept", "application/json");

        yield return request.SendWebRequest();

        if (request.result != UnityWebRequest.Result.Success)
        {
            Debug.LogError("Login failed: " + request.error);
            yield break;
        }

        Debug.Log("Login response: " + request.downloadHandler.text);
    }
}
```

如果是登录后的接口，例如 `GET /api/player/me`，需要加：

```csharp
request.SetRequestHeader("Authorization", "Bearer " + token);
```

## 8. Axios 调用示例

如果前端是 Web，可以这样调用：

```javascript
import axios from "axios";

const api = axios.create({
  baseURL: "https://api.your-domain.com",
  headers: {
    "Content-Type": "application/json"
  }
});

async function login() {
  const res = await api.post("/api/auth/login", {
    mailbox: "test@example.com",
    password: "12345678"
  });

  const token = res.data.data.token;

  const me = await api.get("/api/player/me", {
    headers: {
      Authorization: `Bearer ${token}`
    }
  });

  console.log(res.data);
  console.log(me.data);
}
```

## 9. 前端接入建议

建议前端这样组织：

1. 封装一个统一 `ApiClient`
2. 统一保存 `Base URL`
3. 统一保存 token
4. 对所有鉴权接口自动注入 `Authorization`
5. 对 `401` 做统一处理，跳回登录页
6. 任务系统优先使用 `POST /api/tasks/events`
7. 大页初始化优先使用 `GET /api/game/home`

## 10. 常见问题

### 10.1 前端请求 401

原因：

- 没带 token
- token 已过期
- token 格式不对

处理：

- 检查 `Authorization: Bearer xxx`
- 重新登录

### 10.2 前端请求 403

原因：

- 当前账号没有权限访问该资源

### 10.3 请求成功但数据为空

原因：

- 当前账号没有角色
- 当前角色还没初始化
- 活动当前没有可用配置

### 10.4 AI 接口为什么返回格式不一样

原因：

- `POST /api/campus/qa` 不是统一的 `ApiResponse`
- 这是一个独立格式接口
